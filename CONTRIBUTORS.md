# Contributing

Thanks for wanting to make **Hold My Coffee** better. 🙏

## We accept contributions as issues, not pull requests

This pack is a **curated** set of standards. To keep it coherent and consistent, all changes flow through one editorial pass rather than merged patches. So:

- ✅ **Open an [issue](https://github.com/dylangrech92/hold-my-coffee/issues)** to suggest a change, report a problem, propose a new handbook, or challenge an existing rule (with evidence — that's the whole point).
- ❌ **Please don't open pull requests.** They will be closed unmerged with a pointer back here. The `main` branch is protected and PRs are not part of the workflow.

## What makes a great issue

The skills themselves demand evidence over opinion, so issues do too:

- **The problem, concretely.** What does an agent do today that a senior engineer wouldn't — or vice versa?
- **The evidence.** A transcript, a diff, a reproduction, a citation. "It feels off" is a start; show the moment it went wrong.
- **The proposed change.** Which handbook, which line, what it should say instead.

Good suggestions get folded into the curated text (and you'll be credited in the issue thread).
